# iOS TestFlight Release Guide

This guide covers releasing the Generative Radio iOS app to TestFlight via App Store Connect.

## App details

| | |
|---|---|
| **App name** | Generative Radio |
| **Bundle ID** | `com.generativeradio.app` |
| **Team ID** | `GXWN8P2ZSF` |
| **Xcode workspace** | `mobile/ios/GenerativeRadio.xcworkspace` |
| **Framework** | React Native (New Architecture) |

Always open the `.xcworkspace` file, not `.xcodeproj`.

---

## Step 1 ‚Äî Create the app in App Store Connect (first time only)

1. Go to [appstoreconnect.apple.com](https://appstoreconnect.apple.com) and sign in.
2. **Apps ‚Üí + ‚Üí New App**.
3. Fill in:
   - Platform: iOS
   - Name: Generative Radio
   - Bundle ID: `com.generativeradio.app`
   - SKU: `generativeradio-001` (or any unique string)
4. Click **Create**.

This only needs to be done once.

---

## Step 2 ‚Äî Verify code signing

Open `mobile/ios/GenerativeRadio.xcworkspace` in Xcode.

1. Select the **GenerativeRadio** target ‚Üí **Signing & Capabilities** tab.
2. Ensure **Automatically manage signing** is checked.
3. **Team** should show `GXWN8P2ZSF`.
4. Switch the config dropdown to **Release** and confirm the same settings apply.

If Xcode shows a provisioning profile error, go to **Xcode ‚Üí Settings ‚Üí Accounts**, add your Apple ID, and click **Download Manual Profiles**.

---

## Step 3 ‚Äî Bundle the JS

No manual step needed. The Xcode project has a **"Bundle React Native code and images"** build phase that automatically runs `expo export:embed` during archive. Xcode handles this when you run **Product ‚Üí Archive** in Release configuration.

---

## Step 4 ‚Äî Increment the build number

Each upload to App Store Connect requires a unique build number. The current build number lives in `CURRENT_PROJECT_VERSION` in the Xcode project settings (currently `1`).

To bump it:

- In Xcode: select the **GenerativeRadio** target ‚Üí **General** tab ‚Üí **Build** field.
- Or edit directly in `mobile/ios/GenerativeRadio.xcodeproj/project.pbxproj` ‚Äî search for `CURRENT_PROJECT_VERSION` and increment the value.

The marketing version (`MARKETING_VERSION` / `CFBundleShortVersionString`) only needs to change when you want to show a new version number to users.

---

## Step 5 ‚Äî Archive in Xcode

1. Set the scheme destination to **Any iOS Device (arm64)** ‚Äî not a simulator. The destination picker is in the Xcode toolbar: click the right half of the `[GenerativeRadio] [device name]` selector. Under the **iOS Device** section at the top of the dropdown, select **Any iOS Device (arm64)**. This option requires you to be signed into Xcode with your Apple ID (Xcode ‚Üí Settings ‚Üí Accounts).
2. **Product ‚Üí Archive**.
3. The Organizer window opens when the archive is complete.

---

## Step 6 ‚Äî Upload to App Store Connect

In the Organizer:

1. Select the archive ‚Üí **Distribute App**.
2. Choose **App Store Connect ‚Üí Upload ‚Üí Next**.
3. Leave defaults (strip Swift symbols, upload symbols) ‚Üí **Next**.
4. Choose **Automatically manage signing ‚Üí Next**.
5. Review the summary ‚Üí **Upload**.

Processing takes 5‚Äì15 minutes. You will get an email when it's ready.

---

## Step 7 ‚Äî Set up TestFlight

In App Store Connect, go to your app ‚Üí **TestFlight** tab.

1. Wait for the build badge to go from yellow (processing) to green (ready).
2. Fill in **Test Information** when prompted (what to test, feedback email).

### Internal testers (immediate, no Apple review)
- Up to 100 testers from your App Store Connect team.
- Add under **Internal Testing ‚Üí +**.

### External testers (requires Apple review, ~24‚Äì48h first time)
- Add individual testers or enable a **Public Link**.
- Add under **External Testing ‚Üí +**.

---

## Encryption export compliance

Apple asks about encryption on every first upload. Since the app uses HTTPS (but no non-exempt encryption), this is pre-answered in `mobile/app.json` under `expo.ios.infoPlist`:

```json
"ITSAppUsesNonExemptEncryption": false
```

This propagates into `Info.plist` automatically when Expo generates the native project. Do not edit `Info.plist` directly ‚Äî `mobile/ios/` is gitignored and regenerated by `expo prebuild`.

## Troubleshooting

| Problem | Fix |
|---|---|
| "No provisioning profile" on Release | Xcode ‚Üí Settings ‚Üí Accounts ‚Üí Download Manual Profiles |
| Upload rejected: duplicate build number | Increment `CURRENT_PROJECT_VERSION` and re-archive |
| Build stuck in "Processing" | Wait; if over 30 min, check App Store Connect email for errors |
| JS bundle not found at runtime | Re-run Step 3 before archiving |
| "Upload Symbols Failed" for React.framework / ReactNativeDependencies.framework / hermesvm.framework | **Warnings only ‚Äî not errors.** These prebuilt frameworks ship without dSYMs. Click through and complete the upload; the app will work fine on TestFlight. Crash logs inside those frameworks will show raw addresses rather than function names, which is acceptable. |
| Local network permission dialog at review | Your `NSLocalNetworkUsageDescription` in Info.plist explains this ‚Äî no action needed |
